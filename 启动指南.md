# 配色灵感画廊 - 启动指南

## 项目介绍

这是一个基于React和Vite构建的配色灵感画廊应用，展示了各种预设的配色方案。用户可以点击查看详细的配色卡片。

## 环境要求

- Node.js (推荐 v16 或更高版本)
- npm 或 yarn 或 pnpm

## 安装依赖

在开始之前，请确保您已安装Node.js。然后按照以下步骤操作：

### 方法1：使用命令提示符（CMD）

1. 按下 `Win + R` 键打开运行对话框
2. 输入 `cmd` 并按回车打开命令提示符
3. 导航到项目目录：
   ```
   cd d:\数据存储\配色
   ```
4. 安装依赖：
   ```
   npm install
   ```

### 方法2：使用PowerShell（需要管理员权限）

1. 以管理员身份打开PowerShell
2. 运行以下命令来允许执行脚本：
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```
3. 导航到项目目录：
   ```powershell
   cd d:\数据存储\配色
   ```
4. 安装依赖：
   ```powershell
   npm install
   ```

## 启动开发服务器

依赖安装完成后，可以通过以下命令启动开发服务器：

### 在CMD中：
```
npm run dev
```

### 在PowerShell中：
```powershell
npm run dev
```

## 访问应用

1. 启动开发服务器后，终端会显示一个本地URL，通常是 `http://localhost:5173/`
2. 打开您的浏览器，访问这个URL
3. 您应该能看到配色灵感画廊的界面，展示了各种配色卡片
4. 点击任何卡片可以全屏查看详细信息

## 构建生产版本

如果您想构建生产版本的应用，可以运行：

```
npm run build
```

构建完成后，静态文件将位于 `dist` 目录中。

## 常见问题

### 1. 无法安装依赖或运行npm命令

- 确保Node.js已正确安装：在命令行中运行 `node -v` 和 `npm -v` 检查版本
- 如果遇到权限问题，尝试以管理员身份运行命令提示符或PowerShell
- 可以尝试使用yarn代替npm：安装yarn后运行 `yarn install` 和 `yarn dev`

### 2. 端口被占用

如果端口5173被占用，Vite会自动尝试使用其他端口。如果您想指定端口，可以修改package.json中的dev脚本。

### 3. 应用无法正常显示

- 检查浏览器控制台是否有错误
- 确保所有依赖都已正确安装
- 尝试清除浏览器缓存后重新加载页面
